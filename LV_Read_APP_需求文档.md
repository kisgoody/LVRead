# LV Read APP 需求文档

**版本：** v1.3  
**更新日期：** 2026-07-02  
**状态：** iOS 单端迭代中

---

## 1. 产品概述

### 1.1 产品背景
移动阅读已成为用户日常获取信息和娱乐的主要方式之一。当前市面上的阅读 APP 功能日趋同质化，但在跨设备互通、本地文件管理、个性化阅读体验方面仍有较大提升空间。LV Read APP 定位为一款轻量、高效、注重用户自主管理书籍资源的本地阅读工具，强调设备间便捷的书籍传输与高度可定制的阅读体验。

### 1.2 产品定位
一款以**本地书籍管理 + 沉浸式阅读**为核心的移动端阅读应用，支持多来源书籍导入、同网设备快速传输、丰富的阅读自定义设置，满足用户在无网络或弱网络环境下自主管理阅读内容的场景需求。

### 1.3 产品目标
- 提供流畅、清晰的本地书籍管理体验（增删改查 + 分享）。
- 打造可高度定制化、沉浸无干扰的阅读环境。
- 实现同局域网设备间的书籍快速传输，降低对云服务的依赖。

### 1.4 当前项目总结（2026-07-02）

当前项目为 **iOS 本地阅读器**，主要代码位于 `LVRead/LVRead`。已具备书架、导入、解析、阅读、设置、局域网传输、同网 Web 阅读同步等基础能力。

#### 已实现/进行中的核心模块

| 模块 | 当前状态 | 说明 |
|---|---|---|
| 启动页 | 已实现 | `SplashViewController` 承载开屏动画；开屏完成回调需保证只执行一次 |
| 书架页 | 已实现并持续调整 | 支持书籍列表、搜索、筛选、排序、导入、编辑；当前需求要求顶部品牌标题左对齐，并提升导航栏高度 |
| 文件导入 | 已实现 | 通过系统文件选择、分享入口、同网传输导入书籍 |
| 文件解析 | 已实现基础能力 | 支持 TXT、EPUB、PDF、MOBI/AZW3；不同格式解析质量与兼容性仍需测试集验证 |
| 阅读页 | 重构中 | 重点从“章节驱动”调整为“阅读连续页流 + 按需补页” |
| 翻页动画 | 已实现多模式 | 支持无动画、覆盖、平移、仿真等；动画前必须确保目标页已预载并完成绘制 |
| 滚动阅读 | 重构中 | 需要支持跨章节连续滚动，不能只停留在单章节内容 |
| 页面缓存 | 重构中 | 以当前页为中心的 11 页常驻窗口是当前性能基线 |
| Web 同步阅读 | 已实现基础能力 | 内嵌 HTTP Server + SSE 同步进度与翻页 |
| 主题/阅读设置 | 已实现基础能力 | 支持字体、背景、主题、间距、护眼、水印等设置 |

#### 当前重点问题

1. 书架顶部导航栏需要稳定增高，且品牌标题需左对齐，不允许通过 `layoutSubviews` 反复修改 `frame` 造成布局循环。
2. 阅读页加载逻辑必须避免一次性解析整本书，否则大文件会导致开屏/进阅读页 CPU 飙升与内存暴涨。
3. 阅读页不能只加载一个章节，也不能因为跨章节导致无法翻页或滚动断层。
4. 滑动/翻页时 CPU 和内存不能持续增长，必须有加载去重、缓存窗口上限和视图复用/销毁策略。

### 1.5 最新需求基线（阅读加载与性能）

阅读体验以“顺畅、连续、手势稳定”为第一目标。章节只作为目录跳转、标题展示、进度保存的辅助索引，不应成为翻页/滚动的用户可感知边界。

#### 页面加载规则

| 场景 | 需求 |
|---|---|
| 初始化 | 以当前阅读位置为中心，预加载前后各 5 页，内存常驻目标为 11 页；若当前章节页数不足，应按需加载相邻章节补齐窗口 |
| 普通翻页 | 用户前翻/后翻时，只移动连续页流中的目标页；若目标页不在当前已加载段，后台按方向加载相邻章节并拼接 |
| 缓存平衡 | 后翻补右删左，前翻补左删右；正常阅读过程中内存常驻内容应控制在 11 页附近 |
| 首尾边界 | 全书开头/结尾处不强行补页，不重复创建/销毁页面 |
| 翻页时序 | 页面数据预载完成 → UI 渲染并强制绘制目标页 → 执行翻页动画 |
| 滚动模式 | 滚动触达当前段边界时应按需加载前/后章节并追加到滚动内容，不能只能查看单章节 |
| 页面销毁 | 退出阅读页时，清空阅读页内存缓存、当前书页缓存、临时视图与水印缓存 |

#### 性能约束

| 约束 | 要求 |
|---|---|
| 禁止全书预分页 | 进入阅读页不得遍历并分页整本书；只能加载当前位置附近内容 |
| 禁止重复加载 | 同一方向、同一章节、同一滚动边界的加载请求必须去重 |
| 缓存窗口上限 | 滑动/翻页过程中不得无限追加页面内容；窗口外页面内容应释放或置空 |
| 渲染范围 | 非滚动翻页只渲染当前页与目标页；滚动模式只保留可见区域及预加载区域的页面视图 |
| 监控指标 | 大书阅读过程中 CPU 不应因单次滑动持续满载；内存不应随每次滑动线性增长 |

## 2. 可行性分析

### 2.1 技术可行性：高 ✅

| 维度 | 评估 | 说明 |
|---|---|---|
| 文件解析 | 可行 | EPUB/TXT/PDF/MOBI/AZW3 均有成熟开源库支撑（EPUBlib、PDFBox、MuPDF 等），Android/iOS 双端皆有对应方案 |
| 同网传输 | 可行 | UDP 广播 + TCP 传输方案成熟，Android 有 Nearby Connections API 作备选，iOS 有 MultipeerConnectivity；自定协议开发量中等但可控 |
| 翻页动画 | 可行 | 仿真翻页（OpenGL/Metal 自定义渲染）为技术难点，但社区有大量参考实现；其余四种模式均为标准 View 动画，实现难度低 |
| 本地存储 | 可行 | SQLite/Room + SharedPreferences/DataStore 即可满足，无需服务端 |
| 整体架构 | 可行 | 纯本地应用，无后端依赖，架构复杂度集中在客户端；MVVM + 本地数据库是成熟范式 |

### 2.2 产品可行性：高 ✅

| 维度 | 评估 | 说明 |
|---|---|---|
| 市场定位 | 可行 | 定位清晰——轻量本地阅读器，与 Kindle/微信读书等云阅读形成差异化，目标用户群体明确 |
| 核心价值 | 可行 | "本地书籍管理 + 沉浸阅读 + 同网传输" 三点价值主张明确，解决离线场景与多设备场景的真实痛点 |
| 功能边界 | 可控 | v1.0 MVP 范围合理（书架 CRUD + 基础阅读），未过度设计，可快速验证核心假设 |
| 用户体验 | 有竞争力 | 高度可定制的阅读设置（5 种翻页、自定义字体/背景/间距）是差异化优势 |

### 2.3 资源可行性：中等 ⚠️

| 维度 | 评估 | 说明 |
|---|---|---|
| 开发工作量 | 中等偏大 | 双端（Android + iOS）同时开发，v1.0 MVP 预估 3–4 人月（含测试）；若单端先行则 1.5–2 人月 |
| 技术门槛 | 中等 | 仿真翻页动画、EPUB 解析、PDF 重排、同网传输为技术挑战点，需有相应经验的开发者 |
| 设计资源 | 必要 | 需 UI 设计师产出品牌视觉体系、动效规范、空状态插画等，否则界面质感难以达标 |
| 测试资源 | 必要 | 多格式文件解析、多设备传输兼容性需充分测试，建议建立测试用例库与真机测试矩阵 |

### 2.4 风险与应对

| 风险 | 概率 | 影响 | 应对策略 |
|---|---|---|---|
| PDF 重排版体验不达预期 | 中 | 高 | v1.0 可降级为 PDF 原样渲染 + 缩放平移，重排版放入 v1.2 |
| MOBI/AZW3 解析兼容性不足 | 中 | 中 | DRM 保护文件明确不支持；非标准格式提示用户转换后导入 |
| 同网传输跨平台兼容问题 | 中 | 高 | Android↔iOS 互通需统一传输协议；v1.0 可先做同平台传输，v1.1 补齐跨平台 |
| 仿真翻页性能在低端机不足 | 中 | 中 | 提供"无动画"和"平移"模式作为降级方案；针对低端机自动推荐轻量翻页模式 |
| 大文件（>10MB TXT）加载卡顿 | 高 | 高 | 详见第 11 章《大文件性能处理方案》——子线程解析 + 分批缓存 + 等待窗超时机制 |

### 2.5 结论

项目整体**可行**，技术栈成熟、产品定位清晰、MVP 范围合理。主要风险集中在**大文件性能**和**仿真翻页动画**两个技术点，以及**双端同步开发**的资源投入。建议优先单端（Android）开发验证核心体验，确认产品方向后再补齐 iOS。

## 5. 详细功能需求

### 5.1 书架模块

#### 5.1.1 书籍增加

**功能描述：** 支持三种途径将书籍加入书架。

##### 5.1.1.1 从第三方 APP 分享导入

| 项目 | 说明 |
|---|---|
| 触发方式 | 用户在第三方 APP（如微信、浏览器、文件管理器等）中选择「分享」→ 选择 LV Read，应用接收文件并解析 |
| 支持的格式 | TXT |
| 交互流程 | 1. 系统接收分享 Intent → 2. 自动解析文件元数据（书名、作者）→ 3. 若解析成功，直接加入书架并展示封面；若解析失败，弹出编辑框让用户手动输入书名 → 4. 书架列表刷新 |
| 异常处理 | 文件格式不支持时，弹出 Toast 提示「暂不支持该文件格式」 |

##### 5.1.1.2 从本地文件导入

| 项目 | 说明 |
|---|---|
| 触发方式 | 书架页点击「+」按钮 → 选择「导入本地文件」，弹出系统文件选择器 |
| 支持的格式 | EPUB、TXT、PDF、MOBI、AZW3 |
| 交互流程 | 1. 用户从文件选择器中选取文件（支持单选/多选）→ 2. 应用读取文件并解析元数据 → 3. 自动归类到书架 → 4. 若文件名含特殊字符或重复文件，提示用户确认 |
| 重复处理 | 相同文件（基于文件 hash）已存在时，提示「该书已存在，是否覆盖」，用户确认后替换旧版本 |

##### 5.1.1.3 同网发现设备传输

| 项目 | 说明 |
|---|---|
| 触发方式 | 书架页点击「+」按钮 → 选择「同网传输」，进入设备发现与传输页面 |
| 技术方案 | 基于局域网 UDP 广播自动发现设备 + TCP 传输文件 |
| 交互流程 | 1. 用户点击「同网传输」→ 2. APP 自动扫描同一 Wi-Fi 下运行 LV Read 的设备 → 3. 展示可用的设备列表（显示设备名称、设备型号图标）→ 4. 用户选择目标设备 → 5. 从本地书架勾选要传输的书籍（支持多选）→ 6. 点击「发送」，文件开始传输 → 7. 传输进度以进度条展示 → 8. 完成后双方书架同步刷新 |
| 权限要求 | 需要本地网络权限（Android: INTERNET + ACCESS_WIFI_STATE；iOS: 本地网络权限弹窗） |
| 超时处理 | 设备扫描超时（默认 10 秒）后提示「未发现附近设备，请确保两台设备已连接同一 Wi-Fi」 |

#### 5.1.2 书籍删除

| 项目 | 说明 |
|---|---|
| 触发方式 | 书架页长按书籍 → 弹出操作菜单 → 选择「删除」；或在编辑模式中批量勾选后删除 |
| 交互流程 | 1. 用户触发删除 → 2. 弹出确认对话框「确定删除《书名》？该操作不可撤销」→ 3. 用户确认 → 4. 从书架列表中移除该书目，同时删除 APP 沙盒内对应的本地文件 |
| 批量删除 | 书架页支持进入编辑模式（右上角「编辑」按钮），勾选多本书籍后批量删除 |
| 数据清理 | 删除操作需同步清理：书架数据库记录、本地缓存文件、阅读进度数据 |

#### 5.1.3 书籍修改

##### 5.1.3.1 修改书籍名称

| 项目 | 说明 |
|---|---|
| 触发方式 | 长按书籍 → 选择「修改信息」→ 编辑书名；或在书籍详情页点击书名区域进入编辑 |
| 限制 | 名称不能为空，最长 50 字符，自动去除首尾空格 |
| 交互流程 | 1. 弹出编辑框，预填当前书名 → 2. 用户修改后点击「确定」→ 3. 校验名称合法性 → 4. 更新书架列表显示 |

##### 5.1.3.2 修改书籍封面图

| 项目 | 说明 |
|---|---|
| 触发方式 | 长按书籍 → 选择「修改封面」→ 从相册或系统图库选取图片 |
| 交互流程 | 1. 用户选择「修改封面」→ 2. 弹出系统图片选择器 → 3. 选中图片后裁剪为封面比例（建议 3:4）→ 4. 确认后更新封面显示 |
| 恢复默认 | 提供「恢复默认封面」选项，将封面还原为系统根据书籍信息自动生成的默认封面 |
| 缓存处理 | 修改封面后需清理内存/磁盘中的旧封面缓存 |

#### 5.1.4 书籍查询

| 项目 | 说明 |
|---|---|
| 触发方式 | 书架页顶部搜索栏 |
| 搜索范围 | 按书籍名称、作者名进行模糊匹配 |
| 筛选维度 | 支持按以下维度筛选和排序： |
| | - **名称：** 关键词模糊搜索（实时搜索，输入即查） |
| | - **来源：** 按导入来源分类筛选（分享导入 / 本地文件 / 同网传输） |
| | - **阅读进度：** 全部 / 未读（0%）/ 在读（1%-99%）/ 已读完 |
| | - **分类：** 支持用户自定义分类标签（如 小说 / 技术 / 文学 / 杂志 / 其他），也可按文件格式分类（EPUB / TXT / PDF / MOBI / AZW3） |
| 列表展示 | 书籍卡片需显示：封面图、书名、作者、阅读进度百分比、来源标识 |
| 排序方式 | 支持按 最近阅读 / 最近导入 / 书名（A-Z）/ 文件大小 排序 |
| 空状态 | 搜索无结果时展示空状态插画与文案「没有找到匹配的书籍」 |

#### 5.1.5 书籍分享

##### 5.1.5.1 分享至其他 APP

| 项目 | 说明 |
|---|---|
| 触发方式 | 长按书籍 → 选择「分享」→ 选择「分享到其他应用」 |
| 交互流程 | 1. 调用系统分享面板 → 2. 将书籍文件作为附件传递 → 3. 用户选择目标应用完成分享 |
| 分享格式 | 保持原始文件格式发送；若为 EPUB 格式，可额外提供 TXT 格式选项 |

##### 5.1.5.2 分享至同网发现设备

| 项目 | 说明 |
|---|---|
| 触发方式 | 长按书籍 → 选择「分享」→ 选择「发送到 nearby 设备」 |
| 交互流程 | 与 5.1.1.3 同网发现收发流程一致，从已发现设备列表中选择目标设备进行传输 |
| 传输进度 | 在书架页以悬浮进度条或通知栏形式展示传输进度，支持取消传输 |

### 5.2 阅读模块

#### 5.2.1 目录

| 项目 | 说明 |
|---|---|
| 进入方式 | 在阅读页点击屏幕中央区域 → 底部工具栏弹出 → 点击「目录」图标 |

#### 5.2.4 同网电脑端网页同步阅读

**功能描述：** 用户在手机端阅读时，可一键生成局域网访问链接，在电脑浏览器中打开该链接即可同步查看手机正在阅读的书籍内容。手机翻页时，电脑端页面自动跟随更新，实现「手机握在手，大屏跟着走」的无缝跨屏阅读体验。

##### 5.2.4.1 功能入口与触发

| 项目 | 说明 |
|---|---|
| 触发方式 | 阅读页底部工具栏 → 点击「电脑端阅读」图标（显示器图标），或在设置面板中开启「同网网页同步」 |
| 前置条件 | 手机与电脑连接同一 Wi-Fi；手机端 LV Read 处于前台 |
| 退出条件 | 退出阅读页、APP 退至后台超过 30 秒、手动关闭服务，三者任一满足即自动停止 HTTP 服务 |

##### 5.2.4.2 交互流程

```
手机端操作:
  1. 用户点击「电脑端阅读」图标
  2. APP 在后台启动嵌入式 HTTP Server（端口 8989）
  3. 生成带随机 Token 的访问链接: http://192.168.1.5:8989/?t=a3f8k2
  4. 弹出分享面板，显示链接地址 + 二维码
     ┌──────────────────────────────────┐
     │     📱 电脑端同步阅读            │
     │                                  │
     │   ████████████████████████████    │
     │   ██                      ██     │
     │   ██      [ 二维码 ]      ██     │
     │   ██                      ██     │
     │   ████████████████████████████    │
     │                                  │
     │   链接: http://192.168.1.5:8989  │
     │         /?t=a3f8k2              │
     │                                  │
     │   [ 复制链接 ]  [ 分享链接 ]     │
     │                                  │
     └──────────────────────────────────┘
  5. 用户可复制链接通过微信/QQ 发送到电脑，或电脑扫码直接打开

电脑端操作:
  1. 浏览器打开链接
  2. 页面加载 — 显示当前书籍信息和手机所在阅读页
  3. 电脑端页面布局:
     ┌──────────────────────────────────────┐
     │  📖 《书名》  第3章  进度 37.5%      │ ← 顶部状态栏
     │──────────────────────────────────────│
     │                                      │
     │    （阅读正文内容区域）               │
     │    当前页的完整文本内容...            │
     │                                      │
     │──────────────────────────────────────│
     │  ← 上一页  │  37 / 112  │  下一页 →  │ ← 底部导航
     └──────────────────────────────────────┘
  4. 手机端翻页 → 电脑端自动刷新（SSE 推送）
  5. 电脑端亦可独立翻页浏览（不影响手机端进度）
```

##### 5.2.4.3 同步机制

| 同步方向 | 触发条件 | 实现方式 |
|---|---|---|
| **手机→电脑** | 手机端翻页、目录/进度跳转、修改阅读设置 | SSE (Server-Sent Events) 实时推送，延迟 < 200ms |
| **电脑→手机** | （v2.0 暂不实现双向同步，电脑端独立浏览不影响手机进度） | — |

**SSE 推送事件：**

| 事件类型 | 推送数据 | 触发时机 |
|---|---|---|
| `page_changed` | `{pageIndex, chapterTitle, progressPercent}` | 用户翻页 |
| `position_changed` | `{pageIndex, chapterIndex, chapterTitle, progressPercent}` | 目录跳转或进度条跳转 |
| `settings_changed` | `{fontSize, backgroundColor, nightMode, ...}` | 阅读设置变更 |
| `server_closing` | `{reason}` | 服务即将关闭（退出阅读/后台超时） |
| `heartbeat` | `{timestamp}` | 每 15 秒，保持连接 |

##### 5.2.4.4 安全与隐私

| 措施 | 说明 |
|---|---|
| **随机 Token** | URL 路径中附加 6 位随机 Token（`/?t=a3f8k2`），未知 Token 的请求返回 403。Token 每次启动服务重新生成 |
| **局域网限定** | HTTP Server 仅绑定局域网 IP（非 `0.0.0.0`），蜂窝网络不可达 |
| **内容最小化** | API 默认仅暴露当前阅读窗口内容（当前页 ±5 页），全书不可通过接口遍历获取 |
| **无认证信息** | 不传输任何用户身份、设备信息，仅传阅读状态 |
| **服务生命周期** | 退出阅读页或退后台 30s 后强制关闭 Server，不留常驻端口 |
| **日志脱敏** | 访问日志不记录书籍内容、文件名 |

##### 5.2.4.5 不同文件格式的网页端呈现

| 格式 | 手机端处理 | 网页端渲染 | 备注 |
|---|---|---|---|
| **TXT** | FileParser 按当前排版参数分页 → 输出纯文本 HTML 片段 | CSS 排版渲染（复用手机端字体/行距/配色参数） | 最简路径 |
| **EPUB** | 提取当前章节 XHTML 内容 + 内嵌 CSS 样式 → 整章发送，前端分页 | 浏览器原生渲染 XHTML | 需处理图片等资源相对路径 |
| **MOBI/AZW3** | 先转 EPUB 中间格式 → 同 EPUB 处理 | 同 EPUB | 需集成 Calibre 转换逻辑或预转换 |
| **PDF** | 服务端渲染当前页为 PNG 图片（PDFBox / MuPDF）→ 输出 base64 | `<img>` 标签显示，前后页预加载 | 文本不可选（v2.1 可加 OCR 层） |

##### 5.2.4.6 性能要求

| 指标 | 目标 | 说明 |
|---|---|---|
| HTTP Server 启动 | ≤ 500ms | 点击到弹出链接面板 |
| 首頁加载（电脑端） | ≤ 2s | 含页面内容 + CSS |
| SSE 推送延迟 | ≤ 200ms | 手机翻页到电脑更新 |
| 网页端翻页 | ≤ 300ms | 前后页预加载命中时 < 50ms |
| HTTP Server 内存增量 | ≤ 20MB | 不含页面缓存 |
| 手机端电量影响 | ≤ 3% / 小时 | HTTP Server + SSE 长连接 |

##### 5.2.4.7 技术实现要点

| 要点 | Android 方案 | iOS 方案 |
|---|---|---|
| 嵌入式 HTTP Server | NanoHTTPd（~100KB，单依赖） | GCDWebServer（CocoaPods/SPM） |
| SSE 推送 | `text/event-stream`，`FlushableResponse` | `text/event-stream`，chunked transfer |
| 网页端 | 纯静态 HTML + CSS + vanilla JS（EventSource API），零框架，~15KB gzipped | 同左，网页端跨平台一致 |
| 二维码生成 | ZXing 库（已内置） | CoreImage.CIFilter.qrCodeGenerator（系统 API） |
| 局域网 IP 获取 | `WifiManager.getConnectionInfo()` | `NWPathMonitor` / `NetworkExtension` |

##### 5.2.4.8 异常处理

| 异常场景 | 处理方式 |
|---|---|
| 未连接 Wi-Fi | 提示「请在手机设置中连接 Wi-Fi 后使用此功能」 |
| 电脑端 Token 错误 | 返回 403 + 页面显示「链接已失效，请在手机端重新生成」 |
| 手机端退后台 >30s | 自动关闭 Server；SSE 推送 `server_closing` 事件，网页端显示「手机端已断开」 |
| 手机 IP 变更（DHCP 续租） | SSE 连接断开，网页端显示「连接丢失，请刷新页面或重新扫码」 |
| 端口被占用 | 自动递增端口（8989→8990→8991）并在链接中反映 |
| 电脑端浏览器不支持 SSE | 降级为每 3 秒轮询 `/api/page/current`（网页端自动检测 `EventSource` 可用性） |
| HTTP Server 异常崩溃 | 手机端静默重启（最多 1 次），超出则提示用户手动重试 |

## 6. 非功能需求

### 6.1 性能要求
- APP 冷启动时间 ≤ 2.5 秒
- 书架列表渲染（100 本书以内）≤ 0.5 秒
- 翻页响应延迟 ≤ 50ms
- 同网传输速度 ≥ 2MB/s（同 5GHz Wi-Fi 环境下）
- 阅读页滚动/翻页帧率 ≥ 55 FPS
- APP 内存占用峰值 ≤ 200MB（不含加载中的大文件）

### 6.2 文件解析支持
- **EPUB：** 支持标准 EPUB 2/3 格式，解析 OPF 元数据、NCX 目录、内嵌 CSS 样式
- **TXT：** 支持 UTF-8/GBK/GB2312/Big5 编码自动检测，支持按章节正则分割目录
- **PDF：** 支持 PDF 1.4–2.0，支持重排版（reflow）模式
- **MOBI/AZW3：** 基础解析，提取文本和目录

### 6.3 数据存储
- 书架元数据：使用轻量级本地数据库（SQLite 或 Room）
- 阅读进度：独立存储，支持多本书籍的断点续读
- 用户设置偏好：使用 SharedPreferences / NSUserDefaults 或 DataStore
- 缓存策略：封面图和背景图片使用 LRU 磁盘缓存，上限 100MB

### 6.4 网络与安全
- 同网传输基于局域网，不经过外部服务器，数据不离开本地网络
- 不收集用户阅读行为数据，无需网络账号体系
- 文件传输采用 TCP 协议，支持断点续传（可选 v2.0 实现）
- 不申请不必要的权限（不读取通话、位置、联系人）

### 6.5 兼容性
- **Android：** 最低支持 Android 8.0（API 26），目标 SDK 34+
- **iOS：** 最低支持 iOS 14.0，适配 iPhone + iPad
- 适配全面屏、刘海屏、折叠屏
- 支持横竖屏切换（阅读页默认锁定竖屏，可在设置中关闭锁定）

## 7. 交互与 UI 设计要点


### 7.1 书架页
- 默认以**网格模式**展示书籍封面（2 列/3 列自适应），支持切换为**列表模式**
- 书籍封面图区域显示阅读进度条（底部薄条或右上角百分比角标）
- 长按进入快速操作菜单（修改 / 分享 / 删除）
- 下拉刷新书架、上拉加载更多（大量书籍时分页加载，每页 50 本）
- 支持拖拽排序（用户可自由排列书籍顺序）


### 7.2 阅读页
- 隐藏式 UI：阅读状态下导航栏和工具栏自动隐藏，点击屏幕中央区域呼出
- 点击屏幕左侧区域翻上一页，右侧区域翻下一页（对上下滚动模式禁用此交互）
- 顶部状态栏显示：返回按钮、当前书名、电池电量/时间（可选系统状态栏透传）
- 底部工具栏：目录 / 夜间模式 / 设置 / 自动阅读 / 进度跳转
- 进度跳转：底部显示一个可拖动的进度条和百分比数字，下方标注当前章节名

### 7.3 统一交互规范
- 所有操作按钮建议使用 Lucide 图标 + 长按/悬停显示 Tooltip
- 设置面板采用底部弹出式 Sheet（Bottom Sheet），不跳转新页面
- 所有滑块调节均实时生效，调整即所见
- 删除等不可逆操作必须有二次确认弹窗
- 大量数据刷新时使用骨架屏（Skeleton）而非转圈加载

### 7.4 色彩体系与主题设计（核心视觉规范）

LV Read 的视觉设计必须**打破传统阅读 APP 的沉闷印象**，以鲜亮、突出、个性化的色彩体系建立品牌辨识度。整体色调需做到**第一眼就抓住用户眼球**，同时保持阅读场景的沉浸舒适。

#### 7.4.3 书架页色彩方案——「活力书架」

| FAB 按钮（+） | 大尺寸圆形悬浮按钮，`#7B2FFF` 电光紫底色 + 白色图标，带呼吸脉冲动画，成为书架页最抢眼的操作入口 |

### 7.5 动效与界面设计要求

- **品牌视觉辨识度：** 整体界面突出个性化视觉表现，以珊瑚橙×电光紫双色体系建立独特品牌基因，区分于市面上千篇一律的蓝/灰阅读 APP。
- **全链路动效反馈：** 按钮点击采用弹性缩放（spring animation），状态切换使用渐隐渐显（crossfade），展开收起使用高度动画（expand/collapse），删除操作使用卡片飞出消散效果，让每次操作都有"手感"。
- **页面转场：** 书架→阅读页使用书本翻开式自定义转场（共享元素过渡：封面放大进入阅读页）；阅读页→书架使用合书退出动画；普通页面使用 300ms ease-in-out 滑动转场。
- **微交互：** 下拉刷新使用书本弹跳动画；书籍导入成功使用撒花/粒子效果；传输完成使用脉冲波纹扩散；空状态插画使用循环微动效（呼吸/漂浮）增加生命感。
- **性能红线：** 所有动画帧率 ≥ 55 FPS，复杂动画（仿真翻页）在低端机自动降级为简化版。

#### 10.2.4 同网网页同步 HTTP 接口

> 手机端嵌入式 HTTP Server 对外暴露的 RESTful 接口，供同局域网电脑浏览器调用。基础路径 `http://<phone-lan-ip>:8989`。

##### 接口总览

| 方法 | 路径 | 说明 | 认证 |
|---|---|---|---|
| `GET` | `/` | 阅读器网页 SPA 入口 | URL Token |
| `GET` | `/api/book/info` | 当前书籍元数据 | URL Token |
| `GET` | `/api/page/current` | 当前页内容（HTML 片段） | URL Token |
| `GET` | `/api/page/{n}` | 第 n 页内容 | URL Token |
| `GET` | `/api/chapters` | 章节目录树 | URL Token |
| `GET` | `/api/settings` | 手机端当前阅读设置 | URL Token |

| `POST` | `/api/page/turn` | 遥控翻页（v2.1 可选） | URL Token |

##### 通用规范

| 规范项 | 说明 |
|---|---|
| 认证方式 | 所有请求需携带 `?t=<token>` 查询参数，token 为服务启动时生成的 6 位随机字符串 |
| Token 错误 | 返回 `403 Forbidden` + `{"error":"invalid_token","message":"链接已失效，请在手机端重新生成"}` |
| Content-Type | JSON 接口统一返回 `application/json; charset=utf-8`；页面接口返回 `text/html; charset=utf-8` |
| CORS | 开启 `Access-Control-Allow-Origin: *`，允许任意 Origin 的网页请求 |
| 分页 | 不支持遍历：`/api/page/{n}` 仅返回当前页 ±3 页范围内的内容，超出范围返回 404 |

## 11. 大文件性能处理方案

> 大文件（≥ 5MB 的 TXT，≥ 10MB 的 EPUB/PDF）的导入和阅读是阅读器类 APP 最常见的性能瓶颈。本章定义**导入阶段**和**阅读阶段**的完整性能策略，确保主线程不被阻塞，用户始终感知操作进度。

### 11.1 文件大小分级策略

| 级别 | 文件大小 | 处理策略 |
|---|---|---|
| 🟢 小型 | < 1MB | 主线程直接解析，无感知延迟 |
| 🟡 中型 | 1MB – 5MB | 子线程异步解析，简短 loading 指示器 |
| 🟠 大型 | 5MB – 20MB | 子线程 + 进度等待窗 + 分批处理 |
| 🔴 超大型 | > 20MB | 子线程 + 分块解析 + 进度等待窗 + 超时保护（最大 500MB） |

### 11.2 导入阶段——子线程异步处理

#### 11.2.1 处理流程

```
用户选择文件
     │
     ├── 主线程: 立即关闭文件选择器，显示「导入等待窗」
     │
     ├── 子线程池: 启动导入任务
     │      │
     │      ├── Step 1: 计算文件 SHA-256 → 更新进度 10%
     │      ├── Step 2: 检测编码（TXT） → 更新进度 20%
     │      ├── Step 3: 解析元数据（书名/作者/目录） → 更新进度 50%
     │      ├── Step 4: 复制文件到 APP 沙盒 → 更新进度 80%
     │      ├── Step 5: 写入数据库记录 → 更新进度 95%
     │      └── Step 6: 生成封面图缓存 → 更新进度 100%
     │
     ├── 主线程: 接收进度回调，更新等待窗进度条
     │
     └── 完成: 关闭等待窗，书架列表刷新，显示导入成功动效
```

#### 11.2.2 技术实现要点

| 要点 | 实现方案 |
|---|---|
| **线程模型** | 使用专用后台线程池（`Dispatchers.IO` / `GCD dispatch queue (.userInitiated)`），最多 2 个并发导入任务 |
| **进度回调** | 使用 `Flow` / `Combine Publisher` 发射阶段性进度，主线程 collect 更新 UI |
| **可取消** | 等待窗提供取消按钮，取消时通过 `Job.cancel()` / `NSOperation.cancel()` 中断子线程 |
| **文件复制** | 大文件复制使用 `FileChannel.transferTo()` / `NSFileManager.copyItem` + 64KB 缓冲区流式复制，避免一次性读入内存 |
| **SHA-256** | 分块流式计算（8KB chunk），避免将整个文件加载到内存 |

#### 11.2.3 导入等待窗 UI 规范

### 11.3 阅读阶段——分批缓存页面加载策略

> 核心原则：**永远不要一次性加载整本书的所有页面**。只缓存用户当前阅读位置前后有限窗口内的页面，其余页面按需动态生成并加入缓存。

#### 11.3.1 缓存窗口模型（Sliding Window Cache）

```
                    当前阅读位置（屏幕可见）
                          │
         ┌────────┬───────┴───────┬────────┐
         │ 预加载  │  内存缓存     │ 预加载  │
         │ (前 5 页)│ 当前页      │ (后 5 页)│
         └────────┴───────────────┴────────┘
               │                       │
         LRU 磁盘缓存          待排版队列（子线程）
         (前后各 50 页)         按优先级生成页面
```

| 缓存层级 | 容量 | 说明 |
|---|---|---|
| **L1 内存缓存** | 当前页 ± 5 页（目标共 11 页） | 翻页零延迟，直接渲染 |
| **L2 预加载缓冲** | 仅保存邻近章节的待排版任务，不常驻页面视图 | 当前窗口不足 11 页时按需补齐 |
| **L3 磁盘缓存** | 前后各 50 页 + LRU 淘汰（上限 200 页） | 避免重复排版计算，空间换时间 |

#### 11.3.2 页面加载流程

```
进入阅读页
     │
     ├── 主线程: 显示骨架屏（Skeleton Page）→ 禁止用户交互
     │
     ├── 子线程: 异步加载目标阅读位置
     │      │
     │      ├── 1. 从数据库读取阅读进度 → 定位章节和页偏移
     │      ├── 2. 从文件流式读取当前位置所在章节文本
     │      ├── 3. 对当前页进行排版计算（measure + layout）→ 入 L1 缓存
     │      ├── 4. 预排版当前页前后各 5 页 → 入 L1 缓存
     │      ├── 5. 当前章节不足 11 页时，按需加载相邻章节补齐窗口
     │      └── 6. 通知主线程渲染
     │
     ├── 主线程: 移除骨架屏，渲染当前页
     │
     └── 后台持续: 仅维护当前页前后 5 页窗口（低优先级, IdleHandler 空闲时执行）
```

#### 11.3.3 翻页时的缓存调度

```
用户翻页（下一页）
     │
     ├── L1 命中? → 直接渲染（< 1ms）
     │
     ├── L2 任务命中? → 等待排版完成后提升到 L1，渲染（< 16ms）
     │
     └── L3 命中? → 子线程重新排版验证（设置变更时需重排）后渲染
          同时触发:
            - 页面窗口向后滑动，右侧补 1 页
            - 页面窗口左侧淘汰 1 页
            - 首尾边界不强行补页、不重复创建/销毁页面

翻页方向感知:
  - 向后翻页（下一页）→ 补右删左，窗口维持当前页 ±5 页
  - 向前翻页（上一页）→ 补左删右，窗口维持当前页 ±5 页
  - 跳转（目录跳转/进度条拖拽）→ 清空 L1/L2，目标页变为最高优先级
```

#### 11.3.4 内存保护机制

| 机制 | 阈值 | 行为 |
|---|---|---|
| **L1 上限** | 11 页（首尾边界可少于 11 页） | 超过时立即释放最远页面内容与视图 |
| **L2 上限** | 仅保留当前方向的少量排版任务 | 重复方向、重复章节、重复边界请求必须去重 |
| **内存警告** | `onTrimMemory(TRIM_MEMORY_RUNNING_LOW)` / `didReceiveMemoryWarning` | 清空 L2，L1 收缩到 ±1 页 |
| **总内存硬上限** | 阅读模块 80MB | 超限时强制 GC，仅保留 L1 当前页，其余丢弃 |

#### 11.3.5 主线程防卡死措施

| 措施 | 实现 |
|---|---|
| **排版异步化** | 页面排版（measureText + layout）全程在子线程执行，主线程仅负责 Canvas 绘制 |
| **文本测量复用** | `StaticLayout` / `NSLayoutManager` 实例池化复用，避免频繁创建销毁 |
| **大章节分片** | 单章超过 50 页时，仅排版可见窗口附近页面，其余懒加载 |
| **空闲预加载** | 利用 `IdleHandler`（Android）/ `NSRunLoop` 空闲时机预排版后续页面，不抢占翻页资源 |
| **渲染帧监控** | 翻页动画期间暂停后台预加载，动画完成（`onAnimationEnd`）后恢复 |
| **异常保护** | 单页排版超过 200ms 时取消当前排版任务，降级为纯文本流式渲染（无分页） |

#### 11.3.6 连续跨章节加载策略

```
用户滑动/点击翻页到当前已加载段边界
     │
     ├── 主线程: 保持当前页，暂不执行翻页动画
     │
     ├── 子线程:
     │      ├── 按方向加载相邻章节文本
     │      ├── 排版缺失页面
     │      └── 拼接到连续页流
     │
     ├── 主线程: 目标页完成绘制后执行翻页动画
     │
     └── 超时处理: 3 秒内未完成则显示「加载中...」并允许用户返回上一页
```
