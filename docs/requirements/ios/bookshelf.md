# 书架模块需求（iOS）

## 需求拆解

书架是底部导航第一项，负责本地藏书管理和继续阅读。iOS原生实现必须使用 UIKit。

## 功能范围

| 能力 | 说明 |
|---|---|
|顶部导航栏|左对齐标题和服标题，标题：LVRead，副标题：为文字而生，因阅读而狂热；右侧“+”号导入书籍按钮：本地文件导入、同网传输、第三方分享导入 |
| 继续阅读 | 首屏展示最近阅读书籍的名称、作者、章节、书籍格式、阅读进度 |
| 我的书籍（藏书列表） | 支持网格和列表两种密度，展示书名、作者、格式、进度、来源 |
| 筛选 | 全部、阅读中、待读、已读完、收藏，并添加数量显示 |
| 排序 | 最近阅读、最近导入、书名A-Z、文件大小 |
| 编辑 | 修改书名、删除、分享、批量选择 |


## 页面状态

| 状态 | UI 要求 |
|---|---|
| 有内容 | 全功能 |
| 空书架 | 顶部导航栏除外其他功能都不展示，并添加空状态说明：请点进右上角“+”号导入书籍，或通过第三方APP分享到LVRead |
| 搜索无结果 | 保留搜索词，提示调整关键词或筛选条件 |
| 加载中 | 使用书籍卡骨架屏，不使用单独转圈 |
| 错误 | 展示可恢复错误文案和「重试」按钮 |

## iOS 交互

- Tab 文案固定为「书架」，SF Symbols 使用 `books.vertical`。
- 顶部操作按钮触控区域不小于 44pt。
- 长按书籍弹出 `UIContextMenuInteraction`，包含继续阅读、编辑信息、分享、删除。
- 删除必须二次确认，并同步清理书籍记录、阅读进度、评论、书签和统计索引。
- iPad 横屏时可采用侧边栏 + 详情区布局，书籍列表不应铺满到难以扫读。

## 验收标准

- 0 本书、1 本书、100 本书都能正常布局。
- 超长书名和作者名必须截断或换行，不遮挡按钮。
- 深色模式下卡片、进度条、筛选选中态可读。
- VoiceOver 能读出书名、作者、进度和当前状态。
