# 笔记模块需求（iOS）

## 需求拆解

笔记是底部导航第二项，负责汇总阅读评论和书签。评论与书签都从阅读页产生，笔记页负责统计、筛选、检索和回到原文。

## 评论功能

| 项目 | 说明 |
|---|---|
| 触发 | 阅读页长按当前段落 |
| 选区 | 默认选中当前段落，允许用户调整选区范围 |
| 输入 | 底部 Sheet 输入评论文字，支持取消、保存、清空 |
| 校验 | 评论正文不能为空，最大 1000 字；选中文本最大保存 500 字 |
| 标记 | 保存后阅读页段落侧边显示评论标记，点击可查看或编辑 |
| 回跳 | 笔记页点击评论卡回到对应书籍、章节、页码、段落 |

## 书签功能

| 项目 | 说明 |
|---|---|
| 触发 | 阅读页下拉设置书签 |
| 取消 | 当前页已有书签时再次下拉取消书签 |
| 标识 | 阅读页页角或顶部显示书签标识，不只依赖颜色 |
| 去重 | 同一书籍、章节、页码只允许一个活动书签 |
| 回跳 | 笔记页点击书签卡回到对应阅读位置 |

## 笔记页结构

| 区域 | 内容 |
|---|---|
| 顶部统计 | 评论数、书签数、最近记录 |
| 分段控件 | 全部、评论、书签 |
| 筛选 | 按书籍、时间、关键词 |
| 列表 | 评论卡、书签卡混排或按类型筛选 |
| 空状态 | 无笔记、无评论、无书签、筛选无结果分别展示 |

## 阅读页标记规则

- 评论标记位于段落右侧，不改变正文排版宽度。
- 书签标识位于页面页角或顶部，不遮挡正文。
- 同一段落既有评论又在书签页范围内时，两个标记并列展示，VoiceOver 读出「有评论，有书签」。
- 标记加载失败不影响阅读正文展示，笔记页可提示数据刷新失败。

## 验收标准

- 长按段落、保存评论、回到原文链路完整。
- 下拉设置/取消书签链路完整。
- 无笔记状态能清楚引导用户从阅读页添加。
- 评论和书签删除后统计同步更新。
