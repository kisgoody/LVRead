# LVRead 阅读建议方案（iOS）

## 1. 文档信息

| 项目 | 内容 |
|---|---|
| 方案版本 | 1.0 |
| 适用平台 | iOS |
| 展示页面 | 我的、阅读统计 |
| 数据来源 | 本地阅读统计、分时阅读记录、单本书阅读统计 |
| 建议数量 | 每次最多 2 条 |
| 话术数量 | 19 类建议，每类 5 条，共 95 条 |

## 2. 目标

阅读建议不再根据每日目标生成，而是根据用户真实的阅读统计提供：

1. 一条可执行的阅读建议；
2. 一条阅读习惯发现或正向反馈。

每日目标只负责展示完成进度，不参与阅读建议的类型判断、优先级和话术生成。

## 3. 数据输入

| 数据 | 用途 |
|---|---|
| 最近 30 天每日阅读分钟数 | 周期趋势、活跃日期、连续阅读 |
| 最近 12 周阅读分钟数 | 长周期趋势 |
| 每日 24 小时阅读分钟数 | 时间段、高峰、双高峰、时段迁移 |
| 单本书阅读时长 | 阅读主线、常读书籍 |
| 单本书最后阅读时间 | 阅读中断、最近阅读书籍 |
| 书籍阅读进度 | 同时在读数量 |

当前未完整保存单次阅读会话，因此本方案不生成“单次阅读过短”“阅读过于碎片化”等建议。

## 5. 触发条件

| 建议 ID | 建议类型 | 触发条件 |
|---|---|---|
| `data.insufficient` | 数据不足 | 最近 14 天少于 3 个阅读日或累计不足 60 分钟 |
| `reading.inactive` | 阅读中断 | 连续 3 天以上没有阅读 |
| `trend.decline` | 阅读下降 | 最近 7 天比前 7 天下降 30% 以上，且前 7 天不少于 60 分钟 |
| `trend.growth` | 阅读增长 | 最近 7 天比前 7 天增长 20% 以上，且前 7 天不少于 60 分钟 |
| `days.concentrated` | 日期集中 | 最近 7 天不少于 60 分钟，但只在 1～2 天阅读 |
| `streak.stable` | 连续阅读 | 连续阅读达到 3 天以上 |
| `books.too_many` | 在读过多 | 同时在读超过 3 本 |
| `book.dominant` | 阅读主线 | 某本书占近期阅读时长 60% 以上 |
| `time.overnight` | 凌晨阅读 | 00:00～05:00 占比 30% 以上，至少出现 3 天 |
| `time.early_morning` | 清晨阅读 | 05:00～08:00 符合稳定时段标准 |
| `time.morning` | 上午阅读 | 08:00～11:00 符合稳定时段标准 |
| `time.noon` | 午间阅读 | 11:00～14:00 符合稳定时段标准 |
| `time.afternoon` | 下午阅读 | 14:00～18:00 符合稳定时段标准 |
| `time.evening` | 晚间阅读 | 18:00～21:00 符合稳定时段标准 |
| `time.night` | 夜间阅读 | 21:00～24:00 符合稳定时段标准，且未触发凌晨阅读 |
| `time.dual_peak` | 双高峰 | 两个不相邻时段占比均达到 20% 以上 |
| `time.week_pattern` | 工作日/周末差异 | 两类日期的高峰时段相差 2 小时以上 |
| `time.shift` | 时段迁移 | 最近 7 天高峰比前 7 天移动 3 小时以上 |
| `time.flexible` | 时间分散 | 任意连续 2 小时窗口占比均低于 25% |

### 5.1 稳定时间段标准

具体时间段建议必须同时满足：

- 最近 14 天累计阅读不少于 120 分钟；
- 至少有 5 个有效阅读日期；
- 目标时间段占总阅读时长 35% 以上；
- 目标时间段至少在 3 个不同日期出现。

展示时使用该区间内阅读最多的连续 2 小时，例如清晨区间为 05:00～08:00，最终文案可以显示 06:00～08:00。

## 6. 选择规则

### 6.1 输出限制

- 每次最多展示 2 条建议；
- 最多展示 1 条时间相关建议；
- 同一次不展示 2 条负向建议；
- 数据不足时只展示数据积累提示；
- 趋势增长与趋势下降不能同时出现；
- 每日目标不参与任何建议判断。

### 6.2 行动建议优先级

```text
阅读中断
→ 趋势下降
→ 凌晨阅读
→ 在读过多
→ 阅读日期集中
```

### 6.3 阅读发现优先级

```text
趋势增长
→ 连续阅读
→ 阅读主线
→ 工作日/周末差异
→ 双高峰
→ 时段迁移
→ 具体时间段
→ 时间分散
```

### 6.4 时间建议覆盖关系

```text
凌晨阅读
→ 工作日/周末差异
→ 双高峰
→ 时段迁移
→ 具体时间段
→ 时间分散
```

- 凌晨阅读覆盖普通夜间建议；
- 双高峰覆盖单一时间段建议；
- 工作日与周末差异明显时覆盖普通高峰建议；
- 时段发生明显迁移时不同时展示旧时段建议；
- 无明显时间规律时才使用时间分散建议。

## 7. 动态变量

| 变量 | 含义 |
|---|---|
| `{days}` | 间隔或连续天数 |
| `{current}` | 当前周期阅读分钟数 |
| `{previous}` | 上一周期阅读分钟数 |
| `{percent}` | 周期变化比例 |
| `{range}` | 第一阅读时间段 |
| `{range2}` | 第二阅读时间段 |
| `{oldRange}` | 原阅读高峰 |
| `{newRange}` | 新阅读高峰 |
| `{book}` | 书名 |
| `{count}` | 书籍数量 |

## 8. 话术库

### 8.1 数据不足 `data.insufficient`

1. 再完成几次阅读后，将根据你的阅读节奏生成更准确的建议。
2. 当前阅读记录还比较少，继续阅读一段时间后可以发现你的习惯。
3. 暂时没有足够的数据判断阅读趋势，新的记录会让建议更加准确。
4. 阅读数据正在积累，后续将从时间、频率和书籍偏好分析你的习惯。
5. 继续保持几次自然阅读，系统会逐步识别适合你的阅读节奏。

### 8.2 阅读中断 `reading.inactive`

1. 距离上次阅读已经 {days} 天，可以从最近阅读的书继续。
2. 最近 {days} 天没有阅读记录，不妨先读一个短章节找回节奏。
3. 阅读暂时中断了 {days} 天，可以从上次停留的位置重新开始。
4. 已有 {days} 天没有继续阅读，今天可以安排一次轻量阅读。
5. 最近的阅读节奏有所停顿，继续一本熟悉的书会更容易恢复。

### 8.3 阅读下降 `trend.decline`

1. 最近 7 天阅读 {current} 分钟，比前 7 天减少 {percent}%，可以选择熟悉的时段恢复阅读。
2. 本周阅读时间有所下降，可以从最近在读的书开始一次短阅读。
3. 最近一周比上一周少读了 {percent}%，不妨先恢复最容易坚持的阅读时段。
4. 阅读时间从 {previous} 分钟下降到 {current} 分钟，可以尝试减少同时在读的书。
5. 最近阅读频率有所放缓，选择篇幅较短的章节会更容易重新进入状态。

### 8.4 阅读增长 `trend.growth`

1. 最近 7 天阅读时间比前 7 天增加 {percent}%，当前节奏正在提升。
2. 本周已经阅读 {current} 分钟，比上一周更加稳定。
3. 最近一周的阅读投入明显增加，可以继续保持当前安排。
4. 阅读时间从 {previous} 分钟提升到 {current} 分钟，这个节奏值得继续。
5. 最近的阅读表现比上一周期更活跃，当前习惯正在逐渐形成。

### 8.5 阅读日期集中 `days.concentrated`

1. 本周阅读主要集中在少数几天，分散到更多日期会更容易保持习惯。
2. 最近的阅读时间比较集中，可以尝试在其他日期增加一次短阅读。
3. 你在少数日期完成了大部分阅读，适当分散可以让节奏更加稳定。
4. 本周阅读总量不错，但日期分布较集中，可以增加几个轻量阅读日。
5. 阅读时间集中在 1～2 天，拆分成多次短阅读可能更容易持续。

### 8.6 连续阅读 `streak.stable`

1. 已连续阅读 {days} 天，当前阅读习惯比较稳定。
2. 连续 {days} 天都有阅读记录，这个节奏值得继续保持。
3. 你已经坚持阅读 {days} 天，稳定性正在逐步增强。
4. 最近 {days} 天保持了连续阅读，可以继续沿用当前安排。
5. 连续阅读达到 {days} 天，说明目前的阅读时间比较适合你。

### 8.7 同时在读过多 `books.too_many`

1. 当前有 {count} 本书同时在读，建议优先继续最近阅读的 1～2 本。
2. 在读书籍已经达到 {count} 本，可以先推进进度最高的书。
3. 同时阅读 {count} 本书可能会分散注意力，可以暂时确定一条阅读主线。
4. 当前阅读内容比较分散，建议从最近打开的书中选择一本优先完成。
5. 有 {count} 本书处于阅读中，可以减少切换，让阅读进度更加连贯。

### 8.8 阅读主线 `book.dominant`

1. 最近主要在阅读《{book}》，可以继续保持这条阅读主线。
2. 《{book}》占据了近期大部分阅读时间，是你当前最主要的阅读内容。
3. 最近的阅读重点集中在《{book}》，继续推进会更容易保持连贯。
4. 《{book}》是近期阅读时间最多的书，可以优先完成当前章节。
5. 近期阅读明显偏向《{book}》，你的阅读主线已经比较清晰。

### 8.9 凌晨阅读 `time.overnight`

1. 最近多次在 {range} 阅读，如果条件允许，可以适当提前阅读时间。
2. 凌晨是你近期阅读比较集中的时段，可以留意是否影响第二天的状态。
3. 近期经常在 {range} 继续阅读，可以尝试将部分内容安排到更早的时间。
4. 最近的阅读高峰出现在凌晨，适当提前开始可能更容易兼顾阅读和休息。
5. {range} 出现了较多阅读记录，可以考虑把长时间阅读调整到晚间更早的时候。

### 8.10 清晨阅读 `time.early_morning`

1. 清晨 {range} 是你最稳定的阅读时段，可以继续保持。
2. 你经常在 {range} 开始阅读，清晨已经成为主要阅读时间。
3. 最近的阅读高峰集中在 {range}，这段安静时间可能比较适合你。
4. 清晨阅读出现得比较稳定，可以优先把需要连续理解的内容安排在这里。
5. 从近期记录看，{range} 是你最容易进入阅读状态的时间。

### 8.11 上午阅读 `time.morning`

1. 最近主要在 {range} 阅读，上午是你当前最集中的阅读时段。
2. 你经常在 {range} 保持阅读，可以继续保留这一时间安排。
3. 上午的阅读记录比较稳定，{range} 可能是你注意力较集中的时段。
4. 近期阅读高峰出现在 {range}，适合继续推进当前阅读主线。
5. 从阅读数据看，你在 {range} 更容易保持连续阅读。

### 8.12 午间阅读 `time.noon`

1. 午间 {range} 是你经常阅读的时段，可以继续安排篇幅适中的章节。
2. 最近的阅读主要集中在 {range}，午间已经形成稳定的阅读习惯。
3. 你经常利用 {range} 阅读，可以优先选择容易暂停和继续的内容。
4. 午间阅读记录比较稳定，保持当前节奏有助于增加阅读频率。
5. {range} 是你近期较活跃的阅读时间，可以继续保留这段阅读间隙。

### 8.13 下午阅读 `time.afternoon`

1. 最近主要在 {range} 阅读，下午是你当前较稳定的阅读时段。
2. 你经常在 {range} 继续阅读，可以把这段时间用于推进在读书籍。
3. 下午的阅读记录比较集中，{range} 可能更符合你当前的安排。
4. 近期阅读高峰出现在 {range}，可以继续沿用这一节奏。
5. 从最近的数据看，你在 {range} 更容易留出完整的阅读时间。

### 8.14 晚间阅读 `time.evening`

1. 晚间 {range} 是你最主要的阅读时段，可以优先保留。
2. 最近大部分阅读发生在 {range}，晚间已经形成稳定节奏。
3. 你经常在 {range} 进入阅读状态，可以把当前主线书籍安排在这里。
4. 晚间阅读记录比较稳定，{range} 适合继续推进较长章节。
5. 从近期记录看，{range} 是你最容易保持连续阅读的时间。

### 8.15 夜间阅读 `time.night`

1. 最近主要在 {range} 阅读，睡前是你较稳定的阅读时段。
2. 你经常在 {range} 结束一天的阅读，可以继续保持适合自己的节奏。
3. 夜间阅读主要集中在 {range}，可以优先选择节奏平稳的内容。
4. {range} 是近期阅读最活跃的时间，可以继续作为固定阅读时段。
5. 从最近的数据看，你更习惯在 {range} 安静阅读。

### 8.16 双高峰阅读 `time.dual_peak`

1. 阅读主要集中在 {range} 和 {range2}，可以分别安排不同篇幅的内容。
2. 你在 {range}、{range2} 都有稳定记录，已经形成两个主要阅读时段。
3. 近期阅读呈现双高峰，{range} 和 {range2} 都比较适合你。
4. 阅读时间主要分布在 {range} 与 {range2}，可以根据当天安排灵活选择。
5. 你通常在 {range} 或 {range2} 阅读，这两个时间共同构成了当前节奏。

### 8.17 工作日与周末差异 `time.week_pattern`

1. 工作日主要在 {range} 阅读，周末则偏向 {range2}，可以分别保持两种节奏。
2. 工作日和周末的阅读时间不同，可以按两类日期分别安排。
3. 工作日高峰为 {range}，周末高峰为 {range2}，时间分布比较清晰。
4. 平日更适合 {range}，周末更适合 {range2}，可以继续采用不同安排。
5. 阅读时段会随工作日和周末变化，两种节奏都已经比较稳定。

### 8.18 时段迁移 `time.shift`

1. 最近阅读时间从 {oldRange} 转移到 {newRange}，阅读习惯正在变化。
2. 近期主要阅读时段变为 {newRange}，比之前明显提前或推迟。
3. 阅读高峰从 {oldRange} 移动到 {newRange}，可以观察新时段是否更容易坚持。
4. 最近一周更常在 {newRange} 阅读，与前一周的 {oldRange} 有所不同。
5. 当前阅读时间正在向 {newRange} 集中，可以继续观察这一变化。

### 8.19 阅读时间分散 `time.flexible`

1. 阅读时间比较灵活，目前没有明显固定时段。
2. 近期阅读分布在多个时间段，可以根据每天的空闲时间安排。
3. 阅读时间较为分散，暂时没有形成单一高峰。
4. 当前没有特别集中的阅读时间，灵活安排可能更适合你。
5. 最近在不同时间都有阅读记录，你的阅读节奏偏向灵活型。

## 9. 五次内不重复机制

每种建议独立保存最近 4 次使用的话术编号。

示例：

```text
time.early_morning → [3, 0, 4, 1]
```

下一次从 `0...4` 中排除以上编号，只能选择：

```text
[2]
```

展示后仅保留最新 4 条：

```text
time.early_morning → [0, 4, 1, 2]
```

该算法可以保证任意连续 5 次使用不同话术。

持久化键：

```text
reading_advice_template_history_v1
```

推荐存储结构：

```json
{
  "trend.growth": [0, 3, 1, 4],
  "time.early_morning": [3, 0, 4, 1],
  "streak.stable": [1, 2, 4, 0]
}
```

### 9.1 不消耗随机次数的情况

- 主题切换；
- 屏幕旋转；
- Auto Layout 重排；
- 页面数据未变化时的刷新；
- 前后台切换后恢复同一展示内容。

只有重新生成一次建议展示，才更新对应建议 ID 的话术历史。

## 10. 展示示例

用户最近数据：

```text
最近 14 天阅读：310 分钟
有效阅读日期：9 天
最近 7 天比前 7 天增长：32%
清晨 06:00～08:00 占比：41%
连续阅读：6 天
```

首次展示：

```text
最近 7 天阅读时间比前 7 天增加 32%，当前节奏正在提升。

清晨 06:00～08:00 是你最稳定的阅读时段，可以继续保持。
```

下次生成：

```text
阅读时间从 133 分钟提升到 176 分钟，这个节奏值得继续。

从近期记录看，06:00～08:00 是你最容易进入阅读状态的时间。
```

建议结论保持由数据决定，仅话术在防重复范围内随机变化。

## 11. 验收标准

- 每日目标变化不会改变阅读建议类型；
