# 阅读器稳定分页与拟真翻页设计

## 目标

在不改变现有阅读主题、工具栏、目录和其他翻页模式的前提下：

1. 让正文阅读区域在页面内水平居中，正文采用两端对齐；
2. 消除相邻页面之间缺字、重字以及修改字号后才恢复的问题；
3. 新建独立拟真翻页实现，提供随手势变化的纸张曲面、分层光影、半透明背面、惯性完成和阻尼回弹；
4. 翻页期间预渲染目标页，避免硬切、闪烁和短暂空白。

## 现状与问题

- `PageContainerView` 使用 CoreText 绘制正文，但段落对齐为 `.natural`。
- `ReaderViewController` 与 `ContinuousReaderViewController` 各自维护分页逻辑，无法保证测量参数始终一致。
- 分页阶段根据完整章节计算 CoreText 可见范围，绘制阶段再用截取后的页文本创建新 Frame。页首上下文变化会影响换行和字形组合，使分页范围与实际绘制不一致。
- 现有 `SimulationAnimator` 使用固定横向条带和固定锚点，不能根据触点纵向位置改变卷角姿态；背面主要是纯色覆盖，并非正面内容的半透明镜像。

## 方案

### 统一排版

新建 `ReaderTextLayoutEngine.swift`，集中负责：

- 根据阅读设置创建字体与段落样式；
- 计算与 `PageContainerView` 完全相同的正文矩形；
- 生成统一的属性字符串；
- 使用 CoreText 可见 UTF-16 范围连续分页；
- 校验所有分页范围连续、单调且完整覆盖章节内容；
- 为页面绘制提供同一套排版参数。

正文矩形使用现有百分比页边距，并在页面内保持左右对称，因此阅读区域水平居中。段落使用 `.justified`；最后一行由排版引擎自然左对齐，避免中文正文逐行居中。

分页循环必须满足：

- 第一页 `startCharOffset == 0`；
- 任意相邻页 `previous.endCharOffset == next.startCharOffset`；
- 每页 `endCharOffset > startCharOffset`；
- 最后一页 `endCharOffset == attributed.length`；
- 字符切片使用 UTF-16 安全边界，不拆分组合字符或代理项；
- 无可见字符时返回明确错误或安全兜底页，不静默跳过字符。

`PageContainerView` 不再依据页字符串重新推断分页边界，而是使用统一引擎生成的样式和正文矩形绘制当前页内容。两个阅读控制器均调用统一引擎，移除重复分页计算。

### 拟真翻页

新建 `PaperCurlAnimator.swift`，由 `PageFlipAnimator` 在 `.simulation` 模式下调用。其他模式保持不变。

动画采用 UIKit/CoreAnimation，不引入第三方依赖：

- 翻页开始前同步完成当前页与目标页布局，生成稳定的正面位图快照；
- 将当前页沿翻页方向划分为自适应数量的纵向网格片；
- 横向拖动决定卷曲进度，手指纵向位置决定卷角中心和各片的垂直偏移；
- 每片使用透视矩阵、Y 轴旋转、平移和轻微剪切近似连续纸面；
- 背面使用正面快照的镜像裁片，降低不透明度并叠加纸张底色，使正面文字可隐约透出；
- 卷曲边缘叠加纸面高光、折痕阴影和落在下一页上的投射阴影；
- 目标页始终位于动画层下方，卷起区域实时露出目标页；
- 动画资源统一保存在 `PageFlipState`，完成、取消、控制器释放时都执行幂等清理。

### 手势与物理行为

两个阅读控制器向动画器传递：

- 当前触点；
- 相对起点位移；
- 即时横向速度；
- 页面尺寸与翻页方向。

动画器对位移做橡皮筋约束，避免越界。松手时综合预测进度与速度：

- 位移达到页面宽度的约 24%，或速度达到约 650 pt/s 且方向正确时提交；
- 其余情况使用弹簧阻尼回到初始状态；
- 提交动画持续时间根据剩余距离和松手速度动态计算，并限制在可感知但不拖沓的范围；
- 反向拖动时允许卷曲姿态收回，不锁死为单调进度；
- 动画过程中禁止启动第二次翻页，但保留取消和资源恢复路径。

### 状态与错误处理

- 目标页不存在时不创建动画，并恢复当前页。
- 快照生成失败时使用无闪烁的淡入淡出兜底，并保证回调只触发一次。
- 页面尺寸为零、控制器正在关闭或手势被系统取消时立即清理。
- 设置变化、旋转和动态字体变化都会使分页缓存失效，并使用新尺寸重新分页。
- 低内存或后台切换时清理临时快照和动画层。
- `reduceMotion` 开启时缩短曲面动画，使用轻量过渡但保持分页逻辑一致。

## 测试策略

先写失败测试，再实现生产代码。

### 排版测试

- ASCII、中文、换行、标点、emoji、组合字符的分页范围连续完整；
- 小页面、大字号、极端行距下不出现零长度页；
- 分页后拼接全部页面内容与原章节内容完全一致；
- 分页测量与绘制使用相同的正文矩形和段落样式；
- 页面宽度变化、字号变化后缓存键变化并重新分页；
- 正文矩形左右边距相等，段落为两端对齐。

### 动画测试

- 进度钳制与橡皮筋函数覆盖负值和越界；
- 不足阈值回弹，超过位移或速度阈值提交；
- 反向速度不能错误提交；
- 动画取消、快照失败和正常完成均只回调一次；
- `cleanup()` 可重复调用且不残留图层或错误页面透明度。

### 构建与人工验证

- 运行项目单元测试；
- 对应用 target 执行 Debug 构建；
- 在不同字号、边距、横竖屏下连续翻阅多个章节；
- 慢拖、快速甩动、拖到一半反向、距离不足、系统取消分别验证；
- 检查浅色、深色主题下的纸张背面、文字透影和阴影层级；
- 使用 Instruments 或 Core Animation FPS 检查交互期间是否有明显掉帧。

## 验收标准

1. 阅读区域左右边距对称，正文两端对齐；
2. 任意相邻页的字符范围严格连续，连续翻页不缺字、不重字；
3. 调整字号、页边距、方向后自动重新分页，无需用户通过二次设置恢复；
4. 左右拖拽均能实时改变卷曲程度，触点上下移动会改变卷曲姿态；
5. 背面为正面内容的半透明镜像，不是纯色遮罩；
6. 卷曲处具有高光、折痕阴影和下一页投影；
7. 距离不足自动阻尼回弹，快速滑动具备惯性完成效果；
8. 全程无页面硬切、闪烁、空白帧和动画资源残留；
9. 覆盖、平移、滚动和无动画模式行为不回归。
